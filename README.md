# DIFUSCO Reproduce

A reproduction of [DIFUSCO: Graph-based Diffusion Solvers for Combinatorial Optimization](https://arxiv.org/abs/2302.08224) (Sun & Yang, NeurIPS 2023), applied to the Travelling Salesman Problem (TSP).

## Method Overview

DIFUSCO formulates TSP as a **denoising diffusion process over graphs**. Given a set of city coordinates, the model learns to generate edge probability heatmaps that indicate which edges belong to the optimal tour.

```
Noisy edge labels (x_T)  →  AGNN denoising (12 layers)  →  Clean heatmap (x_0)  →  Greedy decode + 2-opt  →  Tour
```

Key components:

- **Backbone**: Anisotropic Graph Neural Network (AGNN) with timestep conditioning
- **Diffusion**: Categorical (Bernoulli) or Gaussian, with T=1000 steps
- **Inference**: Iterative denoising with cosine schedule, followed by greedy decoding and 2-opt refinement

## Setup

Requires Python 3.12. Uses [uv](https://docs.astral.sh/uv/) for dependency management.

```bash
uv sync
```

## Data

TSP instances use the standard format from [Joshi et al. (2019)](https://github.com/chaitjo/learning-tsp):

```
x0 y0 x1 y1 ... xN yN output t1 t2 t3 ... tN t1
```

Coordinates are in `[0, 1]`, tour indices are 1-indexed. Ground truth tours are generated by the Concorde exact solver. The paper uses **128,000 instances** for TSP-50.

## Usage

### Training (paper settings)

```bash
uv run python -m src.main data_path=data/tsp50_128000_concorde.txt
```

This runs with the paper's full configuration: 256-dim, 12-layer AGNN, categorical diffusion, AdamW with lr=2e-4, cosine annealing, batch_size=16.

### Quick smoke test

```bash
uv run python -m src.main \
  data_path=data/tsp50_tiny.txt \
  model=small \
  training.epochs=2 \
  training.batch_size=4 \
  training.num_workers=0 \
  wandb.mode=disabled \
  inference.inference_steps=2 \
  inference.use_2opt=false
```

### Config overrides

```bash
# Small model for debugging
uv run python -m src.main data_path=... model=small

# Gaussian diffusion
uv run python -m src.main data_path=... diffusion=gaussian

# Override individual values
uv run python -m src.main data_path=... training.lr=1e-4 training.epochs=100

# Disable W&B
uv run python -m src.main data_path=... wandb.mode=disabled
```

## Hyperparameters (Paper Section 4.1)


| Parameter           | Value                   |
| ------------------- | ----------------------- |
| Hidden dimension    | 256                     |
| AGNN layers         | 12                      |
| Diffusion steps (T) | 1000                    |
| Beta schedule       | linear, 1e-4 to 0.02    |
| Optimizer           | AdamW                   |
| Learning rate       | 2e-4 (cosine annealing) |
| Weight decay        | 1e-4                    |
| Inference steps     | 50 (cosine schedule)    |
| Post-processing     | Greedy decode + 2-opt   |


## Tracking

Training metrics are logged to [Weights & Biases](https://wandb.ai):

- `train/loss_step`, `train/loss_epoch` — diffusion loss
- `train/grad_norm` — gradient norm (clipped at 1.0)
- `val/gap_pct` — optimality gap vs Concorde solutions
- `val/pred_tour_length`, `val/gt_tour_length` — tour lengths
